## О проекте

Проект создан для совместной работы над учебными заданиями с системой контроля версий Git. Каждый участник работает в своей директории, а все изменения проходят через процесс code review с использованием Pull Requests на GitHub.

Особенностью проекта является кросс-платформенная разработка: код писался на разных операционных системах (Windows, macOS, GNU/Linux), и для обеспечения единообразной сборки используется система CMake.

## Кросс-платформенность и CMake

### Почему нужен CMake

Разработчики проекта работали на разных операционных системах, что создавало проблему совместимости. CMake решает эту задачу, предоставляя единый способ описания процесса сборки, который работает везде.

CMake (Cross-platform Make) автоматически генерирует файлы сборки для конкретной платформы: Makefile для GNU/Linux, проекты Visual Studio для Windows или Xcode для macOS. При этом исходный код и конфигурация остаются одинаковыми.

В проекте CMakeLists.txt настроен так, чтобы автоматически находить все .cpp файлы в директориях участников, создавать для каждого исполняемый файл и копировать необходимые входные данные в папку сборки. Это обеспечивает, что любой разработчик может склонировать репозиторий и сразу начать работу на своей системе без дополнительных настроек.

## Система защиты кода (CODEOWNERS)

В репозитории реализована система владения через файл `.github/CODEOWNERS`. Каждый участник является владельцем своей директории, и изменения в ней требуют его одобрения при создании Pull Request. 

Прямые коммиты в ветку main запрещены, все изменения проходят через PR с обязательным code review. Это защищает код каждого участника от случайных или несанкционированных изменений.

## Как запустить проект

### Требования

* CMake версии 3.20 или выше
* Компилятор с поддержкой C++17 (GCC, Clang, MSVC)
* CLion (рекомендуется) или любая другая IDE с поддержкой CMake

### Инструкция для CLion

1. Склонируйте репозиторий:
   ```bash
   git clone https://github.com/Vennilay/LearningCPP.git
   ```

2. Откройте CLion и выберите **File → Open**

3. Выберите директорию проекта `LearningCPP`

4. CLion автоматически обнаружит `CMakeLists.txt` и предложит загрузить проект как CMake проект

5. Дождитесь завершения индексации и генерации файлов сборки (это может занять некоторое время при первой загрузке)

6. В правом верхнем углу появится выпадающий список с доступными целями сборки. Каждый .cpp файл из директории homework автоматически становится отдельной целью с именем формата `homework_ИМЯ_HW_НОМЕР_файл`

7. Выберите нужную цель и нажмите кнопку **Run** (зелёный треугольник) или используйте сочетание клавиш **Shift+F10**

8. Программа скомпилируется и запустится. Все необходимые входные файлы из `files/` будут автоматически скопированы в директорию сборки

### Сборка через командную строку

Если вы не используете IDE, можно собрать проект через терминал:

```bash
# Перейдите в директорию проекта
cd LearningCPP

# Создайте директорию для сборки
mkdir build
cd build

# Сгенерируйте файлы сборки
cmake ..

# Соберите проект
cmake --build .

# Запустите нужную программу (пример)
./homework_Vennilay_HW_1_main
```

CMake автоматически скопирует файлы из директории `files/` в папку сборки, поэтому программы смогут найти необходимые входные данные.

**Для Windows:** исполняемые файлы будут иметь расширение .exe

**Для GNU/Linux и macOS:** исполняемые файлы без расширения
